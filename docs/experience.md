# Probell Experience

## Overview

The Probell website is a **guided product experience**, not a static collection of sections.

The goal is to lead the user through a clear journey:

1. Understand the brand
2. See the product
3. Understand the benefits
4. Take action

The experience should feel intentional, not fragmented.

---

## Experience Principles

### 1. Product as the anchor

The product should remain the central focus throughout the experience.

- Visual hierarchy should prioritize the product
- Supporting content should reinforce product understanding
- The user should always know what is being presented

---

### 2. Guided progression

The user should feel guided through the page.

- Each section should build on the previous one
- There should be a clear sense of progression
- The experience should not feel like unrelated blocks

---

### 3. Clarity over interaction

Clarity is more important than interaction.

- Interactions must support understanding
- Avoid unnecessary complexity
- Avoid interactions that distract from the product

---

### 4. Consistency in structure

- Sections should follow predictable patterns
- Layout should feel stable and intentional
- The user should not need to re-learn the interface

---

### 5. Controlled visual rhythm

- Alternate between visual intensity and simplicity
- Use contrast (dark/light, dense/space) intentionally
- Avoid overwhelming the user with too many competing elements

---

## Interaction Philosophy

Interactions should be:

- subtle
- purposeful
- supportive of content

Avoid:

- decorative animation
- excessive motion
- unpredictable behavior

---

## Content Flow (High-Level)

The homepage should generally move through:

1. Introduction (brand + product)
2. Context (why it matters)
3. Exploration (products or variations)
4. Reinforcement (benefits, lifestyle, trust)
5. Action (clear CTA)

This is a flexible structure, not a strict template.

---

## Flexibility Rule

Specific interaction patterns (e.g. sticky elements, scroll-based transitions, animations) are **not locked at this stage**.

These should be:

- explored
- tested
- refined

before being formalized into the system.

---

## UX Goals

The experience should feel:

- clear
- structured
- confident
- product-focused

---

## Anti-Patterns

Avoid:

- disconnected sections with no flow
- overuse of cards and UI elements
- treating the page as a template layout
- prioritizing effects over clarity

---

## Implementation Rule

Before implementing any feature:

- confirm how it supports the experience principles
- avoid locking in specific interaction patterns too early
- prioritize clarity and structure over complexity
